PandaCode CLI Surface

PandaCode v1 is a narrow coding-task executor for three runtimes: Codex through codexctl session, Claude Code through real tmux, and Bamboo as an embedded provider-native coding loop for domestic OpenAI-compatible models.

Codex Runtime implemented

Uses codexctl session start/send/answer/execute/read/interrupt/stop/list. exec and resume call execute so PandaCode behaves as an executor, not only a planner. Structured questions flow through answer. Defaults to gpt-5.5, xhigh effort, and max permission.

Claude Runtime implemented

Starts Claude Code inside tmux with isolated local settings, inline PandaCode-owned hook settings, and an empty strict MCP config. Defaults to opus, max effort, and max permission. It pastes tasks into the TUI, waits for an explicit completion marker, records hook events, and reads visible/log output from tmux. The banned claude -p stream-json path is not used.

Bamboo Runtime implemented

Runs in-process. PandaCode calls the selected provider directly, gives the model read/search/edit/write/bash tools, writes JSONL events and reports under .pandacode/bamboo/runs, and supports compact-aware resume from the previous run id. Defaults to deepseek with high reasoning effort.

Primary Commands

pandacode doctor --json --cd .
pandacode models --json --cd .
pandacode run --runtime auto --model <model> --effort <effort> --permission max --task-file task.md --json --cd .
pandacode run --runtime auto --model kimi-k2.6 --effort high --task-file task.md --json --cd .
pandacode codex exec --task-file task.md --cd .
pandacode codex resume --session latest --task "continue" --cd .
pandacode claude exec --task-file task.md --cd .
pandacode bamboo exec --provider deepseek --model deepseek-v4-pro --effort high --task-file task.md --cd .
pandacode <runtime> answer --session latest --choice 2 --wait --cd .
pandacode claude resume --session latest --task "continue" --cd .

Runtime Commands

Command Purpose Notes
exec Start a coding task and wait for completion. Accepts --task, --task-file, or stdin -.
resume Continue an existing session with another task. Claude restarts through official claude --resume <session-id> when tmux is gone.
answer Answer a runtime prompt waiting for user input. Claude maps to visible AskUserQuestion options; Codex maps --choice N to codexctl session answer --pick N.
status Read current runtime status. Codex reads app-server run state; Claude reads tmux state plus pending hook input; Bamboo reads local run reports.
logs Read runtime output. Use --json for full machine-readable runtime output.
artifacts Show local session metadata and artifact paths. State lives under .pandacode/ in the target workspace.
model Record model and effort for the next turn. --model and optional --effort.
models List model choices for a runtime. Codex delegates to codexctl models; Bamboo returns its built-in domestic model catalog.
interrupt Interrupt an active turn. Claude sends Escape to tmux; Codex calls session interrupt; Bamboo foreground native runs report unsupported.
stop Stop a session or runtime process. Claude kills tmux session; Codex calls session stop; Bamboo foreground native runs report unsupported.
list / doctor Inspect known sessions and runtime availability. Also available globally as pandacode list and pandacode doctor.

Output Contract

exec and resume return compact JSON summaries: state, runtime ids, final message, usage, log path, artifact paths, and short output tails. If a runtime asks for external input, the command returns state: "waiting_for_user" with pending_user_input instead of timing out or failing.

The stable high-level states are completed, waiting_for_user, idle, running, timeout, stopped, blocked, failed, and unsupported.

Execution Mode

Mode Codex Claude Bamboo
max / limited max: --dangerously-full-access; limited: --sandbox workspace-write --approval-policy never. max: --dangerously-skip-permissions; limited: --permission-mode acceptEdits. max: full Bamboo tool loop; limited: blocks background commands, installs, downloads, and recursive/destructive cleanup.
