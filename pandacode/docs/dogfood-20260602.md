## Issues Found And Fixed

- `--cd app --task-file ../task.md` initially failed because task files were
  resolved only from the caller directory. Task files now fall back to resolving
  relative to `--cd`.
- `--json` failures originally emitted plain stderr. Commands with `--json` now
  return a JSON failure envelope.
- Top-level help leaked runtime glue flags such as `--codexctl-bin`,
  `--claude-bin`, `--tmux-bin`, and Claude-only `--visible`. These remain
  available but are hidden from the progressive-disclosure help surface.
- `--runtime auto --session <id>` previously depended on global latest. It now
  resolves concrete session names from local runtime session records.
- Runtime logs now default to redacted structured tails instead of full terminal
  capture/stdout in top-level JSON.
- Bamboo blocked runs now emit exactly one JSON object. The process exits
  non-zero, but `stdout` remains parseable JSON.
- Kimi tool-loop history now includes `reasoning_content` on assistant tool-call
  messages, which is required when K2.6 thinking is enabled.

## Remaining Product Notes

- Bamboo is the most complete runtime for domestic model coding: usage, cache,
  changed files, verification, compact metadata, and artifacts are structured.
- Claude and Codex are usable through the same command shape, but their
  delegated runtimes do not expose Bamboo-level provider cost/cache telemetry.
- Automation should pass explicit session ids. `latest` is kept for human and
  single-task convenience.
